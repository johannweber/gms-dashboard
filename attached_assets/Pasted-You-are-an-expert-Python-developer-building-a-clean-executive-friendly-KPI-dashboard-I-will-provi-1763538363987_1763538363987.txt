You are an expert Python developer building a clean, executive-friendly KPI dashboard.

I will provide you with an Excel workbook that contains our departmental KPI “database”.
Use this spec to build a Streamlit app (MVP) that reads the Excel file and creates a 4-tab interactive dashboard for our Executive Head of Department.

------------------------------------
CONTEXT ABOUT THE EXCEL DATA
------------------------------------
The Excel file has at least one key sheet called:

- "2026 BEPLANNING"

This sheet is our main fact table for KPIs.
Each row is ONE KPI item / project with its own target, output and achievement.

The important columns (Afrikaans labels) on the "2026 BEPLANNING" sheet include:

- "2030 TOEKOMSVISIE"      → strategic pillar
- "2026 FOKUS"             → intermediate focus area
- "ANKERGEMEENSKAP"        → anchor community
- "BESKRYWING VAN PROJEK"  → project / KPI description
- "TEIKEN"                 → numeric target value
- "UITSET"                 → numeric actual output
- "KWARTAAL"               → quarter label (e.g. "K1", "K2", "K3", "K4")
- "PROJEKEIENAAR"          → project owner / person
- "STREEK"                 → region
- "DISTRIK"                → district
- "PROVINSIE"              → province
- "% BEHAAL"               → percentage achieved for that row (per item, already calculated in Excel)

Assume:
- "% BEHAAL" is a numeric percentage per KPI row (e.g. 75 for 75%).
- We do NOT need burndown charts or advanced quarter-logic right now.
- This is an MVP: clean, simple, and performant.

------------------------------------
TECHNICAL REQUIREMENTS
------------------------------------
1. Use Python with:
   - streamlit
   - pandas
   - plotly.express (or Streamlit’s built-in chart functions) for visualizations

2. Load the Excel file once at startup:
   - Use something like:
       df = pd.read_excel("EXCEL_FILE_NAME.xlsx", sheet_name="2026 BEPLANNING")
   - Clean column names minimally if needed (but keep the displayed labels in Afrikaans).
   - Make sure the following columns are treated as numeric:
       "TEIKEN", "UITSET", "% BEHAAL"

3. Derived metrics (calculate in Python, not stored back to Excel):
   For the loaded DataFrame, you will frequently need aggregated measures by group (e.g. per pillar, per province):
   - avg_pct_behaal     = mean("% BEHAAL") for that group
   - count_kpi          = number of rows in that group
   - total_teiken       = sum("TEIKEN") for that group
   - total_uitset       = sum("UITSET") for that group
   - min_pct_behaal     = min("% BEHAAL") for that group
   - max_pct_behaal     = max("% BEHAAL") for that group

   IMPORTANT: For performance comparisons and charts, ALWAYS use avg_pct_behaal (simple average of "% BEHAAL" over items) as the main KPI for now.

4. Global filters:
   At the top of the app (or clearly visible on each tab), create filter widgets that can be reused across tabs:
   - Quarter filter: "KWARTAAL" (multi-select, default = all)
   - Province filter: "PROVINSIE" (multi-select, default = all)
   - 2026 Focus filter: "2026 FOKUS" (multi-select, default = all)
   - 2030 Vision filter: "2030 TOEKOMSVISIE" (multi-select, default = all)

   Implement a helper function that takes the raw df and returns a filtered df based on the current widget selections, so all tabs use consistent filtering.

------------------------------------
APP STRUCTURE – 4 TABS
------------------------------------
Use Streamlit’s tabs feature, e.g.:

tab1, tab2, tab3, tab4 = st.tabs(["Bestuursoorsig", "Strategie", "Geografie", "Projekte & Eienaars"])

All labels shown to the user must be in Afrikaans where possible.

==============================
TAB 1: "Bestuursoorsig"
==============================
Purpose: Executive overview for the department.

Components (working off the filtered df):

1) KPI Cards at the top (use st.columns for layout):
   - Card 1: "Gem. % BEHAAL (Algeheel)"
       - Show the overall average of "% BEHAAL" (avg_pct_behaal) across filtered rows.
   - Card 2: "Aantal KPI-items"
       - Show number of rows (COUNT_KPI) in the filtered df.
   - Card 3: "Beste 2030 Toekomsvissie"
       - Find the 2030 pillar with the highest average "% BEHAAL" and show its name and value.
   - Card 4: "Swakste 2030 Toekomsvissie"
       - Find the 2030 pillar with the lowest average "% BEHAAL" and show its name and value.

2) Bar chart: Performance per 2030 pillar
   - Group by "2030 TOEKOMSVISIE".
   - For each pillar, compute:
       avg_pct_behaal, count_kpi, min_pct_behaal, max_pct_behaal.
   - Plot a bar chart with:
       X-axis: pillar name
       Y-axis: avg_pct_behaal
   - Use tooltips to show count_kpi, min_pct_behaal, max_pct_behaal.
   - Sort pillars by avg_pct_behaal ascending, so weak performers appear on the left.

3) Bar chart: Performance per 2026 focus
   - Group by "2026 FOKUS" similarly.
   - Same metrics and sorting (ascending by avg_pct_behaal).

4) Table: "Swakste 10 KPI-items"
   - Sort the filtered df by "% BEHAAL" ascending and show the top 10 rows.
   - Columns to display:
       "% BEHAAL",
       "BESKRYWING VAN PROJEK",
       "2030 TOEKOMSVISIE",
       "2026 FOKUS",
       "PROVINSIE",
       "PROJEKEIENAAR"
   - Format "% BEHAAL" as a percentage with maybe 1 decimal place.
